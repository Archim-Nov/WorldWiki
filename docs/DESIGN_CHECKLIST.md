# 设计清单（NewWorldWiki）

用于页面迭代与视觉验收，避免遗漏关键体验细节。

## 全局体验
- [ ] 视觉基调与当前风格一致（浅色、克制、自然质感）
- [ ] 页面间差异化明确（布局/节奏/图像比例/模块组合）
- [ ] 头部导航可达性清晰，当前入口与权限流程一致
- [ ] 404/空态/加载态有清晰反馈且不干扰主视觉

## 响应式与可读性
- [ ] 手机端首屏信息完整，不被遮挡
- [ ] 主要标题、摘要与核心图片在 360px 宽度下可读
- [ ] 文本行宽与行距舒适（正文 1.6–1.9 行高）
- [ ] 卡片与网格在 2/3/4 列断点下稳定

## 首页与列表页
- [ ] 首页 3 区块布局节奏平衡（随机展柜/最新英雄/最新国家）
- [ ] 轮播/滑块过渡顺滑，动效不过载
- [ ] 列表页筛选可用且有重置入口
- [ ] 空结果状态有提示 + 返回路径

## 详情页视觉（五类）
- [ ] Hero/Creature 以画像为主视觉
- [ ] Country/Region 以地图或场景为主视觉
- [ ] Story 以封面为主视觉
- [ ] 详情页底部“相关推荐”统一样式且展示 3 项
- [ ] 详情页信息结构清楚（标题、摘要/标签、正文）

## 推荐与内链
- [ ] “相关推荐”来源逻辑正常（自动兜底可用）
- [ ] 内链在富文本中可识别、可点击
- [ ] 推荐卡片点击能正确跳转到对应详情页

## 动效与交互
- [ ] 轮播滑动方向明确，视觉差异明显
- [ ] 点击态/悬停态可感知但不刺眼
- [ ] 动效时长平滑，不出现跳帧或闪烁

## 视觉一致性
- [ ] 色彩体系统一（避免多套主色冲突）
- [ ] 边框/圆角/阴影统一且不过度
- [ ] 图片裁切比例一致（避免拉伸/压缩）
- [ ] 标签与徽标样式一致（大小/对齐）

## 内容规范
- [ ] 中文排版统一（全角标点、引号风格一致）
- [ ] 字段缺失时有占位/提示，不出现空白断层
- [ ] 分类/标签中文显示正确

## 可用性与性能
- [ ] 首屏加载合理（主视觉先出现）
- [ ] 图片懒加载与尺寸控制合理
- [ ] 关键页面无报错日志

## 审核/发布前最后检查
- [ ] 视觉回归：移动端 + 桌面端至少各 1 次
- [ ] 关键入口可达：登录、Profile、Studio
- [ ] 数据完整：至少每类有 1 条可展示内容
